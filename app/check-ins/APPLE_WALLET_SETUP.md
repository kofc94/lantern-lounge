# Apple Wallet Setup

All infrastructure is managed by Tofu in `infrastructure/apple/`. The RSA
private key is generated by Tofu itself — no Keychain, no `.p12`, no manual
key generation needed.

## Two-phase deploy

### Phase 1 — Generate the key and CSR

Fill in your Team ID in `infrastructure/apple/variables.tf` (it's public, not
sensitive — commit it):

```hcl
default = "ABCDEF1234"   # your Team ID from developer.apple.com → Membership
```

No `terraform.tfvars` needed for phase 1.

Then apply:

```bash
cd infrastructure/apple
make build   # tofu init
make deploy  # tofu apply
```

Tofu will:
- Generate an RSA-2048 private key and store it in SSM
- Generate a CSR and **print it as an output**
- Download Apple's WWDR G4 cert and store it in SSM

Copy the `csr_pem` output value (the full `-----BEGIN CERTIFICATE REQUEST-----` block).

### Phase 2 — Get the certificate from Apple

1. Go to <https://developer.apple.com/account/resources/identifiers/list/passTypeId>
2. Click **+** → **Pass Type IDs** → register `pass.org.lanternlounge.membership`
3. Click your new Pass Type ID → **Edit** → **Create Certificate**
4. When Apple asks for a CSR, paste or upload the `csr_pem` output from Phase 1
5. Download the issued `pass.cer` file
6. Convert it to PEM:

```bash
openssl x509 -inform DER -outform PEM -in pass.cer -out pass.pem
```

7. Create `infrastructure/apple/terraform.tfvars` with just the certificate:

```hcl
certificate_pem = <<-EOT
  -----BEGIN CERTIFICATE-----
  ... paste contents of pass.pem here ...
  -----END CERTIFICATE-----
EOT
```

8. Apply again:

```bash
make deploy
```

Tofu stores the certificate in SSM. The Lambda is now fully configured.

### Deploy the check-ins app and frontend

```bash
cd app/check-ins
make build && make deploy

cd app/react-webapp
make deploy
```

## Testing

Sign in at <https://www.lanternlounge.org>, open your membership card, and tap
**Add to Apple Wallet**. On iOS/iPadOS the `.pkpass` opens directly in Wallet.

## Notes

- The private key has `prevent_destroy = true` — Tofu will refuse to destroy it.
  Replacing it would invalidate all existing passes.
- The WWDR G4 cert expires **2030-10-24**. To refresh it, change the
  `triggers` value in `infrastructure/apple/main.tf` and redeploy.
- Pass images are solid-color placeholders. Replace the `_make_png` calls in
  `get_apple_wallet_pass.py` with real artwork when ready — no infra changes needed.
